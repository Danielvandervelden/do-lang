# Improve do-lang skills using skill-creator

## Problem Statement

The do-lang npm package has 7 skill files that were generated by the GSD executor during milestone v1.0. While functional, these skills could benefit from refinement using the `/skill-creator` skill to improve:

- Descriptions and trigger conditions
- Examples and edge case handling
- Error messaging and user guidance
- Consistency across the skill family

The skills to improve are located at `skills/do/`:
1. `do.md` - Main router (37 lines)
2. `init.md` - Workspace/project init (13KB, largest)
3. `scan.md` - Database entry creation (6.7KB)
4. `task.md` - Task creation and refinement (6.4KB)
5. `continue.md` - Task resume (4.1KB)
6. `debug.md` - Debugging workflow (2.4KB)
7. `abandon.md` - Task abandonment (1.2KB, smallest)

The skill-creator skill lives at `~/.claude/skills/skill-creator` and should be used for all modifications per project CLAUDE.md conventions.

## Approach

### Compatibility Contract (non-negotiable)
Preserve for each skill:
- `name` field in frontmatter
- `argument-hint` field (if present)
- `allowed-tools` list
- `@references/...` entry points and paths
- Script interface calls (command names, argument patterns)

### Per-Wave Process
For each wave (skill):
1. **Snapshot**: Copy current skill to `skills/do/<name>.md.bak`
2. **Analyze**: Read skill, check line count, identify improvement areas
3. **Refactor rule**: If skill >500 lines, extract procedures to `references/` first
4. **Improve**: Invoke `/skill-creator` with skill path and goals:
   - Improve description (pushy, trigger-friendly)
   - Explain "why" not rigid MUSTs
   - Add examples where helpful
5. **Eval**: Run 2-3 realistic prompts against old vs new skill
   - Verify triggers correctly
   - Verify command routing works
   - Verify output quality maintained
6. **Size check**: Final skill must be <500 lines; if exceeded, extract to `references/`
7. **Commit or rollback**: If eval passes and size OK, remove backup; if fails, restore and retry

### Priority Order (by complexity)
1. `do.md` - Simple router, quick win
2. `abandon.md` - Small, straightforward  
3. `debug.md` - Medium complexity
4. `continue.md` - Resume logic
5. `task.md` - Core refinement flow
6. `scan.md` - Database operations
7. `init.md` - Most complex, likely needs extraction to references/

## Concerns

- **Backwards compatibility**: Addressed via Compatibility Contract above
- **Reference file dependencies**: Preserve all `@references/...` paths; new extractions use consistent naming
- **Script dependencies**: Scripts in `scripts/` are out of scope; preserve all call interfaces
- **Eval coverage**: 2-3 prompts per skill may not catch all edge cases; prioritize trigger and routing tests
