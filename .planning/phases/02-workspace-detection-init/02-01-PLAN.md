  <action>
Create a Node.js script that performs health checks on existing workspaces (per D-04: review mode checks).

The script should:
1. Accept workspace path as argument
2. Return JSON with health check results

**Health checks to implement:**
- `duplicateIndex`: Duplicate entries in `__index__.md`
- `staleProjects`: Project references where project folder was deleted but database entry remains
- `orphanedEntries`: Database entries in projects/ with no matching repo in github-projects
- `missingAgentsSections`: Required sections missing from AGENTS.md
- `pointerConsistency`: CLAUDE.md, CURSOR.md, GEMINI.md all reference AGENTS.md correctly
- `versionMarker`: Check if "do init completed" marker exists and extract version

**Output format:**
```json
{
  "healthy": true|false,
  "version": "0.1.0"|null,
  "issues": [
    {"type": "duplicateIndex", "severity": "warning", "details": "..."},
    {"type": "staleProjects", "severity": "error", "details": ["project-x"]}
  ]
}
```

Use fs module for file operations. Handle missing files gracefully (report as issues, don't crash).

2. "Where should the database live?" (default: <workspace>/database)

4. "Do you have custom workflows to add? (Jira, CI/CD, etc.)" (optional freeform)

**File creation:**
After gathering answers:
1. Create database folder structure:
   - `<database>/projects/`
   - `<database>/shared/`

2. Read agents-template.md, replace placeholders, write to `<workspace>/AGENTS.md`
3. Read pointer-templates.md, extract and write CLAUDE.md, CURSOR.md, GEMINI.md
4. If user provided custom workflows, append to AGENTS.md

**Health check mode:**
When marker exists:
1. Run workspace-health.cjs script
2. Parse JSON output
3. Report issues with severity levels
