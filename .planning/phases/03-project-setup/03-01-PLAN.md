**Health check function `checkProjectHealth(projectPath)`:**

1. Check `.do/` folder exists - if not, return `{ healthy: false, version: null, issues: [{ type: 'noDotDoFolder', severity: 'error', details: '.do/ folder not found' }] }`

2. Check `.do/config.json` exists and is valid JSON - add `{ type: 'noConfig', severity: 'error', details: 'config.json not found or invalid' }` if missing/invalid

3. Check `config.version` field exists - add `{ type: 'noVersion', severity: 'error', details: 'Missing version field in config.json' }` if missing

4. Check required fields exist (`project_name`, `council_reviews`, `auto_grill_threshold`) - add `{ type: 'missingField', severity: 'warning', details: 'Missing field: <fieldname>' }` for each

5. Check `.do/tasks/` folder exists - add `{ type: 'noTasksFolder', severity: 'error', details: '.do/tasks/ folder not found' }` if missing

6. If `config.active_task` is set, check the file exists at `.do/tasks/<active_task>` - add `{ type: 'staleActiveTask', severity: 'warning', details: 'active_task points to missing file: <filename>' }` if missing

7. Return `{ healthy: !hasErrors, version: config.version || null, issues }`

**CLI handling (copy pattern from workspace-health.cjs):**
- Accept `<project-path>` as first argument
- Support `--help` and `--pretty` flags
- Exit code 0 if healthy, 1 if not

**Export:** `module.exports = { checkProjectHealth }`
