  <action>
Create a CommonJS Node.js script that checks if a project has a database entry.

**Per D-14: /do:task must check for database entry before proceeding**
**Per D-15: If missing, display clear message directing to /do:scan**
**Per D-16: Check path: `<database>/projects/<project-name>/project.md`**

**Logic:**

1. **Find workspace config:**
   - Start from cwd, traverse up looking for `.do-workspace.json`
   - If not found, return error: "Workspace not initialized"

2. **Read workspace config:**
   - Parse `.do-workspace.json`
   - Extract `database` path

3. **Find project config:**
   - Check for `.do/config.json` in cwd
   - If not found, return error: "Project not initialized"

4. **Read project config:**
   - Parse `.do/config.json`
   - Extract `project_name`

5. **Check database entry exists:**
   - Build path: `<database>/projects/<project_name>/project.md`
   - Check if file exists

6. **Return result:**

```javascript
{
  "exists": true|false,
  "project_name": "my-project",
  "expected_path": "/full/path/to/database/projects/my-project/project.md",
  "error": null|"Workspace not initialized"|"Project not initialized"
}
```

**CLI interface:**
- `node check-database-entry.cjs` - check from cwd
- `node check-database-entry.cjs <project-path>` - check specific project
- `node check-database-entry.cjs --pretty` - pretty-printed JSON
- `node check-database-entry.cjs --help` - help text
- Exit code 0 if exists, 1 if missing, 2 if error

**Per D-15: Error message format when missing:**

```
This project needs a database entry before running /do:task.

Expected path: <database>/projects/<project-name>/project.md

Run /do:scan to create the database entry.
```

Print this message to stderr when `--message` flag is provided and entry is missing.

**Export for programmatic use:**
```javascript
module.exports = { checkDatabaseEntry };
```
