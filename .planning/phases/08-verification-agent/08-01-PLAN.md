**Structure (follow stage-execute.md pattern):**

1. **Header section:**
   - Title: "# Verification Stage"
   - Description: "This reference file is loaded by /do:continue when the task is in verification stage."
   - Prerequisites: Active task, stage is `verification`, `stages.execution: complete`
   - Entry conditions: If resuming, pick up from last state

2. **Step V0: Load Task Context (per D-23)**
   - Read Problem Statement, Approach, Concerns, Clarifications sections from task markdown
   - This is the full context to verify against, not just Approach

3. **Step V1: Parse Approach Checklist (per D-24)**
   - Extract discrete steps from Approach section
   - Look for numbered lists (1. 2. 3.) or bullet points
   - Convert to checklist format: `- [ ] Step description`
   - If Approach is prose without clear steps, flag as concern

4. **Step V2: Verify Each Step**
   - For each checklist item, check if it was completed during execution
   - Mark done: `- [x] Step description`
   - Mark incomplete: `- [ ] Step description (INCOMPLETE: reason)`
   - Build verification output for Verification Results section

5. **Step V3: Detect and Run Quality Checks (per D-25)**
   - Read `package.json` in project root
   - Detect scripts matching patterns:
     - Lint: `/^lint/i` (e.g., `lint`, `lint:fix`, `lint:check`)
     - Types: `/^(typecheck|tsc|type-check)/i` (e.g., `typecheck`, `tsc`)
     - Tests: `/^test/i` excluding `/watch/i` (e.g., `test`, `test:unit`)
   - Run each detected check: `npm run <script-name>`
   - Capture exit code and output
   - If no scripts detected, note "No quality check scripts found in package.json"

6. **Step V4: Handle Results (per D-26, D-27, D-28)**
   - Determine Pass/Fail (binary, per D-28):
     - PASS: All checklist items complete AND all quality checks pass
     - FAIL: Any incomplete item OR any quality check failure
   - Write Verification Results section to task markdown (format below)
   - If FAIL due to quality check:
     ```
     Quality check failed: <script-name>
     
     Options:
     1. /do:task "Fix: <brief description of failure>"
     2. Handle manually, then run /do:continue
     
     Choose option (1 or 2):
     ```
     Wait for user response. If 1, display "Run the new /do:task command, then return to verify."
   - If FAIL due to incomplete checklist, display what's missing and stop

7. **Step V5: UAT Flow (per D-29, D-30)**
   - Only reached if Step V4 passes
   - Update frontmatter: `stage: verified`, `stages.verification: in_progress`
   - Generate UAT checklist based on what was actually implemented:
     - Parse Execution Log for files changed
     - Parse Approach for user-observable behaviors
     - Create 3-7 checklist items the user can manually verify
     - Format:
       ```
       Please verify manually:
       1. [ ] <user-observable behavior>
       2. [ ] <another behavior>
       ...
       
       All checks complete? (yes/no)
       ```
   - Wait for user response

8. **Step V6: Completion Flow (per D-31, D-32)**
   - If UAT response is "yes" or "y":
     - Update frontmatter: `stage: complete`, `stages.verification: complete`
     - Read `.do/config.json`, set `active_task: null`
     - Write config.json
     - Display: "Task marked complete. File: .do/tasks/<filename>"
   - If UAT response is "no" or "n":
     - Estimate context usage (heuristic: count Execution Log entries * 750 + Clarification Q&As * 300)
     - If estimated < 80%:
       ```
       UAT failed. What would you like to do?
       1. Loop back to execution (describe what to fix)
       2. Spawn new /do:task for the fix
       
       Choose option:
       ```
       If option 1, update `stage: execution`, `stages.verification: pending`, display "Describe what needs to be fixed:"
       If option 2, display "Run: /do:task \"Fix: <description>\""
     - If estimated >= 80%:
       - Generate handoff prompt:
         ```
         Context at 80%+. Starting fresh session recommended.
         
         Copy this to start a new session:
         ---
         Database: <database-path from .do-workspace.json>
         Project: <project-path>
         Task: .do/tasks/<task-filename>
         
         UAT failed: <summary of what user reported>
         Implementation complete, quality checks pass.
         
         /do:continue
         ---
         ```
       - Display handoff prompt and stop
