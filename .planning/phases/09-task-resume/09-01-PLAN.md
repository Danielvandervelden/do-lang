**Step R0: Resume Check**

Document that this step runs at the start of every stage when resuming via `/do:continue`.

**Step R0.1: Load task markdown**
- Read task file from `.do/tasks/<active_task>`
- Parse YAML frontmatter for: `id`, `description`, `stage`, `stages`, `confidence`
- Extract `## Context Loaded` section paths (parse backtick-wrapped paths)
- Extract `## Clarifications` section (last Q&A entry if present)
- Extract `## Execution Log` section (last entry if present)

**Step R0.2: Detect context state (per D-34)**
- Heuristic: Try to recall if task `description` from frontmatter is known
- Since Claude cannot introspect its context window reliably, always proceed to R0.3
- Skip reload only if context was explicitly loaded earlier in this conversation (e.g., user did `/do:task` in same session)

**Step R0.3: Reload context (per D-33)**
Run `load-task-context.cjs` to get paths:
```bash
node <skill-path>/scripts/load-task-context.cjs "<task-description>"
```

For each path in the result:
1. Check if file exists
2. If missing, add to `stale_refs` list
3. If exists, read file to reload context

Also re-read all paths from the task's "Context Loaded" section.

**Step R0.4: Handle stale references (per D-39)**

If `stale_refs` is non-empty, display blocking prompt:
```
Referenced doc(s) not found:
- <path1>
- <path2>

Options:
1. Continue without them (task markdown has context)
2. Stop and locate the docs

Enter 1 or 2:
```

Wait for user response:
- If "1": Log in Execution Log: "Continued without missing docs: [list]"
- If "2": Display "Locate the missing docs, then run /do:continue again." and STOP

**Step R0.5: Display resume summary (per D-35, D-36)**

Determine last action based on stage:
- Stage `refinement` (grilling): Last Q&A from Clarifications section
- Stage `execution`: Last entry summary from Execution Log
- Stage `verification`: "Verification in progress" or last UAT status

Display resume summary:
```
Resuming: <task-id> (stage: <stage>)
Last action: <summary from last entry>

Continue? (yes/no)
```

Wait for user confirmation:
- If "yes" or "y": Continue to stage-specific logic
- If "no" or "n": Display "Paused. Run /do:continue when ready." and STOP

**Step R0.6: Mid-execution progress check (per D-37, D-38)**

Only for `execution` stage when Execution Log has entries but last Status is NOT "Execution complete":

Parse Approach section into numbered steps.
Parse Execution Log Files entries to identify completed work.
Match completed files to approach steps.

Display progress checklist:
```
Execution paused. Progress so far:
- [x] <completed step from approach, matched to files in log>
- [x] <another completed item>
- [ ] <remaining step from approach>

Continue from here? (yes/no)
```

Wait for user confirmation.
