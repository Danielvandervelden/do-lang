# Tech

Documentation for technical patterns, utilities, and architectural decisions.

## When to Document

Document a pattern when:
- It's project-specific and not obvious from the code
- It involves multiple files working together
- Future developers would waste time figuring it out
- It wraps external libraries with custom conventions

## Format

Each tech doc should include:
- **What**: Brief description of the pattern
- **When**: When to use this pattern
- **How**: Code examples and file references
- **Why**: Rationale for this approach

## Example Topics

- API layer & code generation
- Form system patterns
- State management conventions
- Routing & layout patterns
