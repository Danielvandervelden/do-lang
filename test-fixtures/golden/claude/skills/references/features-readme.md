# Features

Documentation for complex features spanning multiple components/files.

## When to Document

Document a feature when:
- It spans multiple routes, components, or API endpoints
- It has business logic that isn't obvious from the code
- It went through significant iterations worth capturing
- It integrates with external systems

## Format

Each feature doc should include:
- **Overview**: What the feature does for users
- **Architecture**: How the pieces fit together
- **Key Files**: Which files implement the feature
- **API Endpoints**: Backend endpoints used (if applicable)
- **Known Issues**: Current limitations or tech debt
